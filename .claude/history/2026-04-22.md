# 2026-04-22

Soft-failure for missing configured folders.

## Accomplished

- **Missing folders are now non-errors.** An entry in `settings.enabledFolders` whose folder no longer exists on disk used to throw from `HiddenFoldersIndexer.runEnable`, which bubbled up to `startEnableTask` and produced a red "Failed to index …" notice plus a warn-level log. Now:
    - `HiddenFoldersIndexer.pathExistsOnDisk(path)` (new public method) is the single source of truth for "is this folder present at the vault root?" — it reuses `adapter.list('/')` to bypass Obsidian's hidden-path filter.
    - `runEnable` short-circuits with a `debug` log and returns cleanly — it does not add the path to `enabledPrefixes` and does not call `ensurePatched`.
    - `plugin.ts::startEnableTask` / `startRebuildTask` pre-check `pathExistsOnDisk` and return _before_ creating a `Notice`, so the user sees nothing at all when a configured folder is missing.
    - `settings-tab.ts::renderFolderList` no longer prunes missing entries from `enabledFolders`. The config is preserved verbatim.
- Re-indexing on reappearance comes for free: the sync paths (`onload`, plugin toggle, `Rescan hidden folders` command, settings toggle, `updateAllowedExtensions`) all call `runBackgroundSync(settings.enabledFolders)`, which iterates the full configured list and re-attempts each one.

## Tests

- Added `pathExistsOnDisk` coverage (hits, misses, empty path).
- Added `enablePath` coverage for missing folders: resolves silently, does not patch the adapter, is idempotent.
- Full suite: 75 pass / 0 fail.

## Key decisions

- **No folder-creation watcher.** Polling `adapter.list('/')` on a timer or watching the vault root for new dot-folders was rejected — the existing sync triggers (restart, toggle, rescan command) cover the realistic recovery paths and the user confirmed that interpretation in the task brief.
- **Config is preserved, not surfaced.** A missing entry remains invisible in the settings UI (which only lists folders currently on disk). Cleaning it up is a manual action (untoggle when the folder reappears, or edit `data.json`). Showing "missing" entries with a broken toggle was considered and dropped — it adds UI complexity for an edge case.
- **Defence in depth.** Both `startEnableTask` and `runEnable` check existence. The pre-check in the plugin prevents the notice flash; the check in the indexer guards against racy callers and future code paths that bypass `startEnableTask`.
